# 具体框架
 - P: transition kernel, parameterized by θ:  P(s_{t+1}, o_t | s_t, a_t; θ)

  - R: reward function (possibly stochastic), parameterized by θ: r_t ~ R(s_t, a_t; θ)

  - γ: discount factor



Observable RAG state (your original):

  s_t := ( q, H_t, E_t, U_t )

    q   : user question for the episode

    H_t : history (queries issued, thoughts/plans, partial drafts, decisions)

    E_t : evidence set (retrieved passages so far)

    U_t : uncertainty signals available from system (scores, σ-like indicators, self-check outputs)

          Note: U_t can be part of s_t, but Level-2 requires b_t(θ) to represent environment uncertainty.



Hidden environment parameter θ (examples; choose what you model):

  θ may include:

    θ_ret  : retriever quality parameters (success prob, precision/recall, domain mismatch rate)

    θ_llm  : LLM reliability (hallucination rate under low evidence, correctness given evidence)

    θ_noise: corpus noise / contradiction rate

    θ_diff : query difficulty / ambiguity regime

  θ is fixed within an episode (standard Bayesian RL assumption) but unknown to agent.



Belief (agent internal state):

  b_t(θ) := P( θ | history up to time t )

  - This is NOT θ itself; θ does not change.

  - b_t updates when agent observes outcomes from retrieval/answering/etc.



Augmented (Bayes-Adaptive) state:

  s̃_t := ( s_t, b_t )

  - This is the state the policy uses in Level-2 (Bayesian POMDP / Bayes-adaptive view).



Action space (your original, kept):

  a_t ∈ A where:

    A1: Retrieve(query rewrite / new query)

    A2: Select / filter / rerank evidence

    A3: Think (internal planning step; no new evidence)

    A4: Answer (finalize response)

    A5: Abstain / Ask user / Stop



Observation o_t (what you can use to update belief):

  - retrieval results: doc scores, overlap, relevance estimates, novelty, contradictions

  - answer feedback: judge score / EM/F1 / self-consistency / verifier outcome

  - consistency signals: evidence support, citation coverage, contradiction detection

  - any proxy for "retriever succeeded" or "LLM is reliable" given current evidence



############################################

# 1) Environment Dynamics (P and R)

############################################



True but unknown transition (environment-side):

  (s_{t+1}, o_t) ~ P( · | s_t, a_t; θ )



Examples mapped to your RAG framework:

  - If a_t = Retrieve:

      retriever returns ΔE_t (random; quality depends on θ_ret, θ_noise, θ_diff)

      E_{t+1} = E_t ∪ ΔE_t

      H_{t+1} = H_t + [issued query / rewrite]

      U_{t+1} = update uncertainty signals from retrieval stats

      o_t     = retrieval outcome summary (e.g., top-k relevance indicators, novelty, contradictions)

  - If a_t = Select/Filter:

      E_{t+1} = g(E_t) (rerank / drop noise); stochasticity can reflect uncertain relevance estimation

      H_{t+1} = H_t + [filter decision]

      o_t     = post-filter evidence diagnostics

  - If a_t = Think:

      H_{t+1} = H_t + [plan/thought]; E unchanged

      o_t     = introspection / self-check outputs (if any)

  - If a_t = Answer/Stop:

      terminal state reached; o_t = judge/verifier feedback (if available)



True reward (environment-side):

  r_t ~ R( s_t, a_t; θ )

  Typical design:

    - terminal: correctness / judge score / EM/F1

    - shaping: -cost_per_retrieve, +citation_quality, -hallucination_penalty, etc.

  Risk-sensitive variants change objective (agent side), not the environment reward definition.



############################################

# 2) Belief Update (Agent side)

############################################



Bayesian update rule (generic):

  b_{t+1}(θ) ∝ P( o_t | s_t, a_t; θ ) * b_t(θ)



Interpretation:

  - b_t is your probabilistic estimate of θ given what you have observed so far.

  - π does not directly set b_t; π chooses actions → actions produce observations o_t → update b_t.

  - This is the source of "information value": some actions generate informative o_t.



Practical note:

  - In code, you rarely maintain a full continuous belief over high-dimensional θ.

  - Common approximations:

      (a) discrete regimes: θ ∈ {good_retriever, bad_retriever} with categorical belief

      (b) conjugate priors for success probability (Beta-Bernoulli): θ_ret = p_success

      (c) amortized belief via a neural network encoder producing belief parameters

      (d) particle filters over θ (particles + weights)



############################################

# 3) Q(s,b,a) and Decision-Making

############################################



Important: "Q(s_t,b_t,a)" is NOT the policy's total E[R] over all time.

It is the expected *action-value* from taking action a at time t, given current state+belief.



Full (sequential) Bayes-adaptive action-value:

  Q( s̃_t, a ) =

    ∫  E[ r_t + γ V(s̃_{t+1}) | s̃_t, a, θ ]  b_t(θ) dθ



Where:

  s̃_t       = (s_t, b_t)

  s̃_{t+1}   = (s_{t+1}, b_{t+1})  and b_{t+1} depends on o_t

  V(s̃)      = max_{a'} Q(s̃, a')   (in optimal control; or computed under π for evaluation)



Myopic (one-step) simplification (often used as heuristic):

  Q_myopic(s̃_t,a) =

    ∫ E[ r_t | s_t, a, θ ] b_t(θ) dθ



Decision rule (greedy w.r.t Q):

  a_t = argmax_a Q(s̃_t, a)



Stochastic decision (softmax / ε-greedy):

  π(a | s̃_t) ∝ exp( Q(s̃_t,a) / τ )

  or ε-greedy for exploration at the policy level



Key conceptual link:

  - b_t determines how you compute the expected return of each action (via the integral).

  - Different b_t → different Q values → different action choice.

# 4) Full Interaction Loop (Bayesian RAG Control)

############################################



Initialize episode:

  θ ~ P0(θ)            # environment draws hidden parameter (or set by reality)

  b_0(θ) = prior       # agent prior over θ

  s_0 = (q, H_0=∅, E_0=∅, U_0=init_uncertainty_features)

  s̃_0 = (s_0, b_0)



For t = 0..T:

  # --- Step A: policy selects action given augmented state ---

  a_t ~ π( · | s_t, b_t )      # policy uses belief-aware state



  # (Optional explicit Q-based selection, if you implement planning)

  # for each candidate action a:

  #   compute Q(s_t,b_t,a) via belief-weighted expectation (and possibly future V)

  # a_t = argmax_a Q(s_t,b_t,a)



  # --- Step B: environment responds (true dynamics depend on θ) ---

  (s_{t+1}, o_t, r_t, done) ~ EnvStep(s_t, a_t; θ)

    where:

      - s_{t+1} updates q/H/E/U per your original RAG transition logic

      - o_t is observation signal used for belief update

      - r_t is reward (shaping + terminal if done)

      - done indicates terminal (Answer/Stop or max steps)



  # --- Step C: belief update (agent inference) ---

  b_{t+1}(θ) ∝ Likelihood(o_t | s_t, a_t; θ) * b_t(θ)



  # --- Step D: advance ---

  s_t = s_{t+1}

  b_t = b_{t+1}



  if done: break



Return trajectory τ:

  τ = { (s̃_t, a_t, r_t, o_t) }_{t=0..T}



############################################

# 5) Where π → trajectory distribution → reward distribution → J(π) lives

############################################



Given a policy π, it induces a distribution over trajectories:

  τ ~ P(τ | π, prior b_0, environment θ)



Why π changes trajectory distribution:

  - π chooses actions a_t based on (s_t,b_t)

  - actions affect what observations o_t you see (especially via Retrieve)

  - observations update b_{t+1}, which changes future decisions

  - hence the entire (s̃_0, a_0, o_0, b_1, a_1, ...) distribution changes with π



Trajectory distribution → reward distribution:

  - rewards r_t depend on environment outcomes (retrieval success, correctness, etc.)

  - thus different trajectories yield different reward sequences



Policy objective (risk-neutral):

  J(π) = E_{τ~P(·|π)} [ Σ_t γ^t r_t ]



Risk-sensitive alternatives (change objective, not P itself):

  - Utility-based:

      J_U(π) = E[ U( Σ γ^t r_t ) ]

  - CVaR tail-risk:

      J_CVaR(π) = -CVaR_α( Σ γ^t r_t )

  These alter which π is optimal because they score the reward distribution differently

  (average vs tail outcomes).



############################################

# 6) Optimization / Learning (e.g., PPO) — Agent-side, not part of P

############################################



We cannot enumerate all π, so we optimize π with RL algorithms (PPO etc.).



Training loop (high-level):

  repeat:

    collect rollouts τ_i by running current π in the RAG environment (with belief updates)

    estimate advantage / returns from τ_i (risk-neutral or risk-sensitive variant)

    update policy parameters to increase J(π)

  until convergence / budget



Key point:

  - PPO optimizes π to maximize J(π).

  - b_t is part of the agent’s internal state used during rollouts; π is trained to map (s,b) → a.

  - θ is never changed by training; θ only determines environment responses in each episode.

# 7) Mapping back to your original RAG control diagram (explicit)

############################################



Your original:

  s_t = (q, H_t, E_t, U_t)

  a_t ∈ {Retrieve, Filter, Think, Answer, Stop}

  P(s_{t+1}|s_t,a_t) defined by retriever/LLM dynamics

  r_t = correctness - cost - hallucination penalty + citation quality shaping

  objective max E[Σ γ^t r_t]



Bayesian upgrade (Level-2):

  - Introduce θ so true transition is P(s'|s,a;θ)

  - Maintain belief b_t(θ)

  - Augment state to s̃_t=(s_t,b_t)

  - Define Q(s̃_t,a) using belief-weighted expected returns

  - Policy becomes π(a|s̃_t)

  - π affects observations → belief evolution → trajectory distribution → reward distribution → J(π)



############################################

# 8) Clarifications that resolve the confusions we discussed

############################################



(1) Does b_t "determine" P?

  - No. True P is determined by θ (environment).

  - b_t only determines the agent's *estimate* / expectation over outcomes,

    which affects action selection.



(2) Does π "change" θ?

  - No. θ is fixed (standard assumption).

  - π changes what you observe, so it changes b_t (your knowledge), not θ.



(3) Why do exploration-like strategies exist?

  - Because actions can have information value: they change future belief b_{t+1},

    improving future decisions and thus future rewards.

  - The goal is still maximizing J(π), not estimating θ as an end in itself.



(4) Is Q(s_t,b_t,a) "E(R)"?

  - It is an *action-specific* expected return from taking a now (possibly including future),

    computed by integrating over θ with belief b_t.

  - J(π) is the policy-level expected discounted sum over entire trajectories.

# 离线在线设计思路
+ **离线（offline）**：不和真实当前环境交互，用已有数据、已有轨迹、已有标注来训练/建模/初始化。
    - 语料库 / 索引构建
        * 文档清洗 
        *  chunking 
        *  embedding 
        *  FAISS / BM25 index 建立 
        *  多路 retriever 准备
    - 初始 reward 设计 / 打分器设计
    - belief 模块的先验 / 估计器预训练
        * 用历史数据训练一个 evidence sufficiency classifier 
        *  训练 retrieval quality estimator 
        *  训练 hallucination risk predictor 
        *  初始化 b0(θ)b_0(\theta)b0(θ) 的 prior
    - 用已有数据做 SFT / imitation / offline RL warm start
    - benchmark 切片和 failure-mode 标注
+ **在线（online）**：policy 当前选动作，环境当前返回 observation / reward，agent 再根据新反馈继续决策或更新
